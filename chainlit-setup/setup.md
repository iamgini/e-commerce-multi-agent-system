# Chainlit Setup

## Data Persistence

Chainlit offers an official datalayer which can be appended to langgraph for storing sessions and conversation history. However the current implementation (2.10.1) is bugged.

### How to fix

Replace the existing chainlit_data_layer.py binary file in the package with the merged PR copy. For details of the fix, refer to: https://github.com/Chainlit/chainlit/pull/2855/changes

***Note: This issue has been resolved as of 2.11.0.***

## Implementation

To implement the official chainlit-datalayer, begin by adding python dependenices. Only the ones to be used are required to be added into code environment.

```shell
# For the database.
uv add asyncpg

# For cloud providers
uv add boto3                       # AWS S3
uv add azure-storage-blob aiohttp  # Azure
uv add google-cloud-storage        # Google Cloud
```

Add the following environment variable into .env file in the root folder. Again, only the ones to be used are required to be added into code environment. This step must be done first as it is required for prisma to imprint the database.

```shell
# To link to the PostgreSQL instance.
DATABASE_URL=postgresql://<USERNAME>:<PASSWORD>@<IP>:<PORT>/<DATABASE>?sslmode=disable

# S3 configuration.
BUCKET_NAME=my-bucket
APP_AWS_ACCESS_KEY=random-key
APP_AWS_SECRET_KEY=random-key
APP_AWS_REGION=eu-central-1
DEV_AWS_ENDPOINT=http://localhost:4566

# Azure Blob Storage configuration.
BUCKET_NAME=my-container
APP_AZURE_STORAGE_ACCOUNT=devstoreaccount1
APP_AZURE_STORAGE_ACCESS_KEY=Eby8vdM02xNOcqFlqUwJPLlmEtlCDXJ1OUzFT50uSRZ6IFsuFq2UVErCz4I6tq/K1SZFPTOtr/KBHBeksoGMGw==
APP_AZURE_STORAGE_CONNECTION_STRING=DefaultEndpointsProtocol=http;AccountName=devstoreaccount1;AccountKey=Eby8vdM02xNOcqFlqUwJPLlmEtlCDXJ1OUzFT50uSRZ6IFsuFq2UVErCz4I6tq/K1SZFPTOtr/KBHBeksoGMGw==;BlobEndpoint=http://localhost:10000/devstoreaccount1;QueueEndpoint=http://localhost:10001/devstoreaccount1;TableEndpoint=http://localhost:10002/devstoreaccount1
DEV_AZURE_BLOB_ENDPOINT=http://localhost:10000/devstoreaccount1
```

Setup the PostgreSQL service if not done yet. Once the service is running, imprint the prisma schema to it.

```shell
# Setup the postgres docker compose file and create postgres container
docker compose up -d

# Create python virtual environment
cd ~
python -m venv .venv
.venv\Scripts\activate

# Python dependencies for prisma imprint
pip install --upgrade pip setuptools
pip install prisma

# If met with - error while loading shared libraries: libatomic.so.1
# Install missing linux dependencies
sudo dnf install libatomic

# With python environment containing prisma activated
prisma migrate deploy --schema /path/to/schema.prisma
```

## Enable Chat History

By default, chainlit allows users to search, browse and resume their past conversations. However, this feature requires 2 requisites:

- Enable data persistence (see above)
- Configure authentication

### Enabling authenication

To add authentication to chainlit, add the authentication callback decorator into the frontend code. For more information, refer to: https://docs.chainlit.io/authentication/overview

For authenciation to work, a **CHAINLIT_AUTH_SECRET** environment variable must be added to the .env file. This secret can be generated by using the create-secret command

```shell
# Create python virtual environment
python -m venv .venv
.venv\Scripts\activate

# Python dependencies for chainlit
pip install --upgrade pip setuptools
pip install chainlit

# Generate secret
chainlit create-secret
```

### on_chat_resume

Decorator to enable users to continue a conversation. By default, the decorator will automatically:
- Send the persisted messages and elements to the UI
- Restore the user session

For more details, refer to: https://docs.chainlit.io/api-reference/lifecycle-hooks/on-chat-resume


## Translations & Formatting
If the following error was encountered:

```
Translated markdown file for <language> not found. Defaulting to chainlit.md.
```

It means that the chainlit translation and welcome page markdown documents are missing from the app and falling back to defaults (en-US).

### How to fix

1) Add the missing (language).json file into the translations folder
2) Create a dedicated markdown file for the language

### Post-fix folder structure

```shell
/home
├── .chainlit              
│    ├── translations
│    │    ├── en-US.json
│    │    ├── zh-CN.json
│    │    └── add language.json file here
│    │    
│    └── config.toml
│
├── chainlit.md
└── add chainlit_language.md file here
```
